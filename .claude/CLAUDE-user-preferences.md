# User Memory - Personal Preferences

## Code Style
- Prefer explicit over clever
- No emojis in code or documentation

## Git Workflow
- Create atomic commits
- Amend small fixes (spelling, formatting) into main commit
- Build incremental PRs that stack
- Use Co-Authored-By: not adding Generated in git commit messages.

## Pull Request Template
1. **Summary**: Brief overview
2. **Business Justification**: 2-3 sentences on operational impact
3. **Technical Details**: Implementation specifics if complex
- "Write self documenting code and do not rely on code comments, minimize adding comments where possible."
